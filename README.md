## Требования

- Python >= 3.14
- [uv](https://docs.astral.sh/uv/) для управления зависимостями

## Установка

```bash
make sync
```

## Запуск лабораторных

```bash
make run                 # список доступных лабораторных
make run lab01           # запустить конкретную лабораторную
make run lab02_random
```

## Тесты

```bash
make test
```

## Документация

Сайт собирается из каталога [docs/](docs/) через [MkDocs](https://www.mkdocs.org/) с темой [Material](https://squidfunk.github.io/mkdocs-material/) и автоматически публикуется на GitHub Pages при пуше в `main` (workflow [.github/workflows/docs.yml](.github/workflows/docs.yml)).

```bash
make docs                # локальный сервер на http://127.0.0.1:8000
make docs-build          # собрать статический сайт в site/
```

## Лабораторные

### lab01 — массивы и стек

Целочисленный массив со случайным заполнением, функции вывода по индексу, поиска максимума (по значению и индексу) и суммы элементов без встроенных функций, а также стек на основе массива.

### lab02 — связный список, очередь, бинарное дерево

Односвязный список с добавлением/удалением/печатью, очередь (FIFO) на его основе и бинарное дерево поиска. `print_tree` рисует дерево сверху вниз с ветками `/` и `\`.

### lab02_random — случайные данные и сбалансированное дерево

Список и очередь заполняются случайными значениями. Метод `build_balanced` строит дерево из отсортированного массива «серединой вперёд», поэтому его форма не зависит от порядка ввода. Выводятся два дерева: до сортировки (форма зависит от ввода) и после (сбалансированное).

### lab03 — сортировки и подсчёт перестановок

Массив из 10 случайных чисел и три сортировки сравнением: выбором (в том числе через отдельную функцию поиска индекса минимума `get_index_min_element`), вставками и пузырьковая. Каждая возвращает отсортированный массив и число выполненных перестановок. Меню позволяет запускать любую сортировку, сравнить все алгоритмы по числу перестановок и перегенерировать массив.

### lab04 — линейный и бинарный поиск

Массив из 10 случайных чисел и поиск в нём двумя способами. Поиск простым перебором представлен четырьмя функциями: есть ли элемент (true/false), индекс первого вхождения / −1, список индексов всех вхождений / −1, количество вхождений / 0. Бинарный поиск работает по отсортированному массиву (сортировка вставками переиспользуется из lab03). Отдельный пункт меню формирует отсортированную последовательность из вводимых с клавиатуры чисел через сортировку вставками, и по ней выполняется бинарный поиск.

### lab05 — hash map и разрешение коллизий

Хеш-таблица «с нуля» (без встроенного `dict`) целиком про коллизии. Собственные хеш-функции для чисел и строк (полиномиальный хеш), индекс ячейки как `hash(key) % capacity`. Три способа разрешения коллизий: метод цепочек (`HashMapChaining` — бакет-список пар) и открытая адресация (`HashMapOpenAddr` — линейное пробирование и двойное хеширование) с корректным удалением через надгробие (tombstone). Считаются коллизии и коэффициент заполнения, при превышении порога таблица расширяется вдвое с перехешированием. Меню демонстрирует, что происходит при коллизии ключей, и срабатывание авторасширения.
